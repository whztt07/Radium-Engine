23 // --> Interface that must be implemented by a RenderTechnique
24 // --> depthAmbiant Pass
25 // This pass must compute the Z of each Fragment and, optionally, the ambiant term
26 // (see DepthAmbiantPass shader of Forward Renderer)
27 // --> opaqueLighting Pass
28 // This pass must compute the lighting of the opaque aspect of the material
29 // (see BlinnPhong shader of Forward Renderer)
30 // --> transparentLighting Pass
31 // This pass must compute the lighting of the transparent aspect of the material
32 // (see LitOIT shader of Forward Renderer)
33 //
34 // --> modify RenderObject so that it is able to activate the required technique for rendering
35 // This is done but not yet finalized ... main rendering (internal render) is fixed, secondary
36 // renderings (ui,
37 // debug, ...) still have the possibilities to bypass the notion of Technique"
38 //
39 // A RenderTechnique correspond to a set shader configuration allowing to manage a Material while
40 // rendering.
41 // The set of configurations must contains what is required to render objects in the following
42 // ways :
43 // 1- depth and ambiant-environment lighting :
44 // Required for the depth pre-pass of several renderers.
45 // Must initialise the color. Initialization at 0 or to the base color of the fragments
46 // resulting from ambiant-Environment lighting
47 // * Default/Reference : DepthAmbiantPass shaders
48 // 2- Opaque lighting : THIS ONE IS MANDATORY, EACH MATERIAL MUST AT LEAST BE RENDERABLE AS
49 // OPAQUE OBJECT
50 // Main configuration, computes the resluting color according to a lighting configuration.
51 // The lighting configuration might contains one or severa sources of different types.
52 // * Default/Reference : BlinnPhong shaders
53 // 3- Transparent lighting :
54 // Same as opaque lighting but for transparent objects
55 // * Default/Reference LitOIT shaders
56 // 4- WhatElse ????
57 //
58 /* Exemple of use from Forward Renderer
59  *
60  * depthAmbiant pass
61  * Build a RenderParameters object defining the ambiant lighting configuration (envmap or
62  * irradiancemap or constant factor of base color. for each RenderObject call
63  * ro->render(ambiantLigthingPtarams, renderData(stange name for matrices + time),
64  * TECHNIQUE_DEPTH_AMBIANT);
65  *
66  * opaqueLighting pass :
67  * For each light sources set (from 1 to N light sources)
68  * Build a RenderParameters object defining the lighting configuration.
69  * For each RenderObject
70  * call ro->render(ambiantLigthingPtarams, renderData(stange name for matrices + time),
71  * TECHNIQUE_LIGHT_OPAQUE);
72  *
73  * transparentLighting pass
74  * For each light sources set (from 1 to N light sources)
75  * Build a RenderParameters object defining the lighting configuration.
76  * For each RenderObject
77  * call ro->render(ambiantLigthingPtarams, renderData(stange name for matrices + time),
78  * TECHNIQUE_LIGHT_TRANSPARENT);
79  *
80  * Each technique must be configured with its own shaders.
81  *
82  * TODO : Default rendertechnique must be renderer dependant ...
83  */
